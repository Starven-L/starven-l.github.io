1. Windows下通用数据保护方案—DPAPI

Windows系统下，为了使开发者可以实施针对用户身份上下文加密的方案，系统向开发者提供了一个强大的数据保护API——DPAPI，开发者使用该 API 加密的数据在解密时会使用用户身份的上下文解密，使得该数据仅可被当前用户解密

也就是说：不同的用户，有不同的MasterKey，拿到某个用户的MasterKey之后只能解密对应用户的加密数据dpdata

  • 密文：dpdata
  • 密钥：masterKey
  • 唯一标识符：guidMasterKey

2. DPAPI介绍 - 敏感信息保护

DPAPI (Data Protection API) 从Windows 2000开始引入，MSDN中举例DPAPI可以用来保护的数据有：

  • Web page credentials (for example, passwords)
  • File share credentials
  • Private keys associated with Encrypting File System(EFS), S/MIME, and other certificates
  • Program data that is protected using the CryptProtectData function

4.2 sekurlsa::dpapi

Sekurlsa模块中的功能都是通过操作LSASS进程内存实现的

调用该模块功能时都需要调用一个通用的初始化函数kuhl_m_sekurlsa_acquireLSA

该函数会从LSASS进程中读出一些必要信息【所以是需要elevate和DebugPrivilege权限的】

当LSA以PPL保护运行时，还需要使用诸如PPL Killer来关闭才能够正常获取LSASS进程句柄【★★★★★】

注意：LSASS（Local Security Authority Subsystem Service，本地安全机构子系统服务）LSA（Local Security Authority，本地安全机构）

名称 全称 作用 运行形式 为什么重要
LSA Local Security Authority 核心安全逻辑，验证凭据、生成令牌 系统组件（逻辑层） 所有安全认证的中枢
LSASS Local Security Authority Subsystem Service 承载 LSA 的进程，存储凭据 进程（lsass.exe） 凭据都在内存里
PPL Protected Process Light 防止低权限进程访问敏感进程 内核保护机制 阻止直接 dump LSASS
