k8s sidecar容器 & app主容器 & init容器的关联与区别以及安全问题

发表于2025-07-04|更新于2025-07-04
|浏览量:

Reference

https://kubernetes.io/docs/concepts/workloads/pods/sidecar-containers/

What is Sidecat？

官方文档提到一个特性【FEATURE STATE: Kubernetes v1.33 [stable] (enabled by default: true)】

通常，一个 Pod 中只有一个应用容器

Web 应用程序需要的本地 Web 服务器就是 sidecar

Sidecar 容器是与主应用程序容器一起运行在同一 Pod 中的辅助容器。这些容器用于通过提供其他服务或功能（如日志记录、监控、安全或数据同步）来增强或扩展主应用程序容器的功能，而无需直接更改主应用程序代码

Sidecar containers in Kubernetes

文档写到：Kubernetes implements sidecar containers as a special case of init containers; sidecar containers remain running after Pod startup. This document uses the term regular init containers to clearly refer to containers that only run during Pod startup.

注意看这一句sidecar containers as a special case of init containers，也就是说后面都会着重介绍一下这二者的区别，special到底特殊在哪？

有个Feature Gate 叫 SidecarContainers 如果启用了的话（该功能从 Kubernetes v1.29 开始默认启用）

那么你就可以为 initContainers 里的某些容器指定 restartPolicy（重启策略）了

  • 这些支持重启的 sidecar（附属）容器，虽然写在 initContainers 字段中，
  • 但它们的行为已经独立于其他 init container 和主容器了。

也就是说：

  • 它们不像传统的 init container 那样“执行一次然后退出”，
  • 它们可以一直运行、可以被重启，就像主容器那样【这也就是sidecar的特性】
  • 这些restartable sidecar init 容器可以独立启动、停止或重启，
  • 不会影响同一个 Pod 中的主容器或其他 init container。

Deployment中带Sidecar容器的示例

一个包含两个容器的 Deployment 示例，其中一个是 sidecar

Sidecar容器的生命周期与特性

一个 init 容器其 restartPolicy 设置为 Always，它将在 Pod 的整个生命周期内启动并保持运行

1. init 容器的本质

  • 只要你在 initContainers 中加上 restartPolicy: Always，这个容器就会像 sidecar 一样一直运行；
  • 它不再是传统“执行完即退出”的 init container，而是一个可以陪跑整个 Pod 生命周期的组件；
  • 非常适合跑那些必须早于主容器启动、但又需要长期运行的组件，比如日志代理、前置认证服务、监控探针等。

2. init 容器支持 readinessProbe

如果想等这些容器 ready 后再让 Pod 对外暴露服务，可以在这个参数这里操作

3. sidecar-style 容器不会阻塞后续 init 容器

  • 普通 init container 启动 -> 执行完 -> 退出，才进入下一个；
  • 而 sidecar-style init container 一旦启动（started=true），就进入下一个，无需等它退出；

Jobs with sidecar containers

在 Job 场景下，Job 的完成只取决于主容器的退出状态，不受 sidecar-style 容器是否还在运行的影响。

Differences from application containers

from ChatGPT整理

特性 Application Container Sidecar Container Init Container
启动顺序 与 Pod 启动 与主容器一起或前后启动 必须在主容器启动前按顺序启动
生命周期 持续运行 持续运行 执行完毕即退出
是否支持重启 ✅ ✅ ❌（失败重试，但成功后退出）
是否支持 probe ✅ ✅ ❌
是否可与主容器通信 ✅ ✅ ❌（仅通过共享卷）
改变镜像是否重启整个 Pod 是 否，只重启该容器 无意义（执行完即退出）
用途示例 应用主服务 日志采集、代理、配置服务 数据准备、授权校验、预热任务

Sidecar 容器是 Kubernetes 中一种非常灵活的容器角色，它既不像主容器负责核心业务，也不像 Init 容器那样一次性运行，而是“持久并行协助主容器的工作”，同时具有自己的独立生命周期、重启机制、运行逻辑，适合构建微服务架构中“模块化、组合式”的服务组件。

一句话总结

The sidecar container shares the same lifecycle, resources, and network namespace as the main container.
